14.2. JSON Data¶

JavaScript Object Notation (JSON) is a popular format for exchanging data on the web. This plain text format has a simple and flexible syntax that aligns well with Python dictionaries. It is easy for machines to parse and people to read.

Briefly, JSON has two main structures, the object and the array.

  • object - an unordered collection of name-value pairs that are contained in curly braces, where each name and value are formatted as “name”:value and separated by commas.

  • array - an ordered collection of values contained in square brackets, where the values are separated by commas and unnamed.

The values in an object and array can be of different types and can be nested. That is, an array can contain objects and vice versa. The primitive types are limited to: string, in double quotes; a number in text representation; logical as true or false; and null.

The following short JSON file demonstrates all of these syntactical features. It is a simplification of the JSON format provided by Kiva, a global microfinance organization. The excerpt below contains information about one of the founders.

{"lender_id":"matt", 
 "loan_count":23,
 "status":[2, 1, 3], 
 "sponsored": false, 
 "sponsor_name": null,
 "lender_dem":{"sex":"m","age":77 } 
}

Here we have an object that contains six name-value pairs. The values are heterogeneous; four are primitive values, string, number, logical, and null. Of the other, one nests an array and the other an object within the object. The status array consists of three (ordered) numbers, and the lender_dem object contains demographic information.

The built-in json package can be used to work with JSON files in Python. For example, we can load this small file into a Python dictionary.

import json
from pathlib import Path

kiva_file_path = Path() / 'data' / 'js_ex' / 'kiva_1.json'

kiva_dict = json.load(open(kiva_file_path))
kiva_dict

{'lender_id': 'matt',
 'loan_count': 23,
 'status': [2, 1, 3],
 'sponsored': False,
 'sponsor_name': None,
 'lender_dem': {'sex': 'm', 'age': 77}}

The dictionary matches the format of the Kiva file. This format doesn’t naturally translate to a data frame. The json_normalize method can organize this semi-structured JSON data into a flat table.

kiva_df = pd.json_normalize(kiva_dict)
kiva_df
lender_id loan_count status sponsored sponsor_name lender_dem.sex lender_dem.age
0 matt 23 [2, 1, 3] False None m 77

Notice the third element in this one-row data frame is a list. Whereas, the nested object was converted into two columns.

type(kiva_df.iloc[0, 2])

list

There’s a tremendous amount of flexibility in how data can be structured in JSON, which means if we want to create a data frame from JSON content, we need to understand how the data arre organized in the JSON file. We provide three structures that translate easily into a data frame in the next example.

14.2.1. Example: Air Quality Data Exchange¶

The list of PurpleeAir sites used in the case study in Chapter 12 was JSON formatted. In that chapter, we didn’t call attention to the format and simply read the file contents into a dictionary with the json library’s load method and then into a data frame. We have simplified that file while maintaining the general structure so it coincides with a 4 x 3 data frame to make it easy to print the file contents.

We begin with an examination of the original file, and then reorganize it into two other JSON structures that might also be easily read into a data frame to show the flexibility of JSON. The diagram in Figure 14.2 gives a picture representation of these possibilities.

Fig. 14.2 This diagram of shows different approaches for a JSON-format to store a data frame.¶

The leftmost data frame in the diagram shows an organization by rows. Each row is an object of named values where the name corresponds to the column name of the data frame. Rows would then be collected in an array. This structure coincides with that of the original file. Let’s print the file contents for comparison. (See :numrf:`Chapter %s <ch:files>` for more on finding a path name to a file and printing its contents).

from pathlib import Path
import os

epa_file_path = Path() / 'data' / 'js_ex' / 'epa_row.json'

epa_file_path.read_text().split('\n')

['{',
 '  "Header": [',
 '    {',
 '      "status": "Success",',
 '      "request_time": "2022-12-29T01:48:30-05:00",',
 '      "url": "https://aqs.epa.gov/data/api/dailyData/...",',
 '      "rows": 4',
 '    }',
 '  ],',
 '  "Data": [',
 '    {',
 '      "site": "0014",',
 '      "date": "02-27",',
 '      "aqi": 30 ',
 '    },',
 '    {',
 '      "site": "0014",',
 '      "date": "02-24",',
 '      "aqi": 17',
 '    },',
 '    {',
 '      "site": "0014",',
 '      "date": "02-21",',
 '      "aqi": 60',
 '    },',
 '    {',
 '      "site": "0014",',
 '      "date": "01-15",',
 '      "aqi": null',
 '    }',
 '  ]',
 '}',
 '']

We see that the file consists of one object with two elements, named Header and Data. The Data element is an array with an element for each row, and as described earlier each is an object. We now load the file into a dictionary and check its contents.

f = open(epa_file_path)
data_row = json.load(f)
data_row

{'Header': [{'status': 'Success',
   'request_time': '2022-12-29T01:48:30-05:00',
   'url': 'https://aqs.epa.gov/data/api/dailyData/...',
   'rows': 4}],
 'Data': [{'site': '0014', 'date': '02-27', 'aqi': 30},
  {'site': '0014', 'date': '02-24', 'aqi': 17},
  {'site': '0014', 'date': '02-21', 'aqi': 60},
  {'site': '0014', 'date': '01-15', 'aqi': None}]}

Each element in the Data array coincides with a row in our schematic (leftmost portion of the diagram in Figure 14.2.) And a row is an object.

data_row["Data"][0]

{'site': '0014', 'date': '02-27', 'aqi': 30}

We can quickly convert the array of objects into a data frame with the following call.

pd.DataFrame(data_row["Data"])
site date aqi
0 0014 02-27 30.0
1 0014 02-24 17.0
2 0014 02-21 60.0
3 0014 01-15 NaN

The middle portion of the diagram takes a column approach. Here the columns are provided as arrays and organized in an object where the names match the column names. We have created a file to demonstrate the concept.

epa_col_path = Path() / 'data' / 'js_ex' / 'epa_col.json'
epa_col_path.read_text().split('\n')

['{"site":[ "0014", "0014", "0014", "0014"],',
 ' "date":["02-27", "02-24", "02-21", "01-15"],',
 ' "aqi":[30,17,60,null]',
 '}',
 '',
 '']

‘pandas’ had a read_json method that we can use to load the file straight into a data frame without needing to first load it into a dictionary.

pd.read_json(epa_col_path)
site date aqi
0 14 02-27 30.0
1 14 02-24 17.0
2 14 02-21 60.0
3 14 01-15 NaN

Lastly, we organize the data into a structure that resembles a matrix (on the right in the diagram) and separately provide the column names for the features. The matrix is organized as an array of arrays.

epa_mat_path = Path() / 'data' / 'js_ex' / 'epa_val.json'

epa_mat_path.read_text().split('\n')

['{"vars":["site", "date", "aqi"],',
 '"data":',
 '[["0014", "02-27", 30],',
 ' ["0014", "02-24", 17],',
 ' ["0014", "02-21", 60],',
 ' ["0014", "01-15", null]',
 ']',
 '}',
 '']

We can pass these two elements to create the data frame with the contents of data and name the columns with vars.

data_mat = json.load(open(epa_mat_path))
pd.DataFrame(data_mat["data"], columns=data_mat["vars"])
site date aqi
0 0014 02-27 30.0
1 0014 02-24 17.0
2 0014 02-21 60.0
3 0014 01-15 NaN

With this example, we hope to have gotten across the versatility of JSON in formatting files. The first of these is a pared down version of the file available online from PurpleAir. The Kiva sample file is also a pared down version of a file available on line, and the NetCDF file from the previous section is an actual file from the online Climate Data Store. So much of data is available online that we present next some of the basic notions for getting data from the web.
